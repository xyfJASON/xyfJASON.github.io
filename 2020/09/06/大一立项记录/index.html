fasttext 文本分类

背景

任何一个评论都可以分为 $5$ 类：与商品质量无关、是水军、情感正向、情感中性、情感负向。于是，我们只需将一个评论分为这 $5$ 类之一，这就是一个文本分类问题。

查询资料后了解到，$\textbf{fasttext}$ 是一个精度较高、速度极快的文本分类工具，故采用之。

基本逻辑

标注的 $\text{tag}$：$-1$ 为水军，$0$ 为与质量无关，$1$ 为差评，$2$ 为中评，$3$ 为好评。

训练集、验证集、测试集数据分布如下：

数据集 -1 0 1 2 3 total
训练集 $605$ $467$ $605$ $602$ $8721$ $11000$
验证集 $83$ $141$ $124$ $90$ $2562$ $3000$
测试集 $108$ $83$ $168$ $55$ $2470$ $2884$

训练过程与测试结果

根据训练集和验证集自动调参 $5\text{min}$，得到最终模型作为分类器。

测试了 $2884$ 组数据，准确率 $\color{blue}{88.28\%}$

问题

自己写的一堆差评都标注的 $3$，即好评。

那为什么我们的准确率还高达 $88\%$ ？原因分析：如果我们全部标 $3$，那么准确率就是 $2470/2884=\color{red}{85.64\%}$，这本来就很高啊！经过查找资料，我知道了这个问题叫做样本不均衡。样本不均衡使得分类模型容易过拟合，难以运用到新的数据上，故产生了上述差评标注为好评的现象。这个 $88\%$ 的准确率根本不具有普适性。

样本均衡处理

解决样本不均衡问题的一个最为简单的方法之一是过采样(oversampling)，即重复对小类进行采样。

数据集 -1 0 1 2 3 total
训练集 $8721$ $8721$ $8721$ $8721$ $8721$ $43605$
验证集 $2562$ $2562$ $2562$ $2562$ $2562$ $12810$
测试集 $2470$ $2470$ $2470$ $2470$ $2470$ $12350$

自动调参训练出分类器，测试 $12350$ 组数据，准确率 ${\color{blue}{70.69\%}}$

自然，过拟合问题也得到了初步解决：

进一步思考

思考1：一句评论中会有很多无关紧要的词汇信息，如果能把它们去除，也许能提升分类效果。

思考2：水军和无关言论中出现的一些情感的正向/负向词汇，会对真正需要标注的数据分类造成干扰。

对于思考1：jieba分词+特征降维

背景

针对上述思考1内容，我们利用工具对评论进行分词处理，将无关紧要的词汇去除。

基本逻辑

分词采用 $\textbf{jieba}$ 中文分词工具，停用词表采用网络上的哈工大停用词表，如此可得到一条评论的若干词汇，称为特征。

当前特征过多，可能包含许多不必要信息（噪音），特征降维即减少特征数目，只保留信息丰富的特征。这里采用最简单的降维方式：统计词频，只保留词频大于某个阀值的词汇。

训练过程与测试结果

根据训练集和验证集自动调参 $5\text{min}$，得到最终模型作为分类器。

阀值 特征词数量 准确率
$100$ $269$ $\color{blue}{70.60\%}$
$10$ $1405$ $\color{blue}{73.40\%}$
$5$ $2111$ $\color{blue}{73.07\%}$
$3$ $2751$ $\color{blue}{75.02\%}$
$2$ $3391$ $\color{blue}{74.21\%}$
$0$ $7851$ $\color{blue}{73.67\%}$

结果和查到的一些资料差不多（取阀值为 $2\sim3$ 为最佳）

可以看见准确率略有提升（ $5\%$ ）

验证思考2：3分类

背景

为了验证上述思考2内容（而不是提出解决方案），我试着剔除掉水军数据和无关数据，再次训练看看效果。

基本逻辑

和 $5$ 分类的相同。

标注的 $\text{tag}$：$1$ 为差评，$2$ 为中评，$3$ 为好评。

训练过程与测试结果

  • 无分词：测试了 $3825$ 组数据，准确率 $\color{blue}{80.75\%}$
  • 分词：取阀值为 $4$，共 $2276$ 个特征词，测试了 $3822$ 组数据，准确率 $\color{blue}{82.44\%}$

可以看见，准确率大幅上升了，说明我们的猜测，即“水军和无关言论中出现的一些情感的正向/负向词汇会对真正需要标注的数据分类造成干扰”具有一定的道理。

$\Large\textbf{但是还没有想到合适的办法解决该问题。}$

fasttext(3阶段)

背景

考虑如何解决猜测中的问题。既然判断是否与商品质量有关和判断是否水军对真正的评价的情感分析有影响，那么我们就将它们分阶段训练。

基本逻辑

stages 训练集 验证集 测试集 total
stage1 $(401,9599)$ $(137,2863)$ $(107,2777)$ $15884$
stage2 $(510,9490)$ $(147,2853)$ $(98,2141)$ $15239$
stage3 $(571,445,8984)$ $(162,136,2702)$ $(100,80,1304)$ $14484$
总 $(41,46,64,86,763)$ $1000$

同样的进行过采样：

stages 训练集 验证集 测试集
stage1 $(9599,9599)$ $(2863,2863)$ $(2777,2777)$
stage2 $(9490,9490)$ $(2853,2853)$ $(2141,2141)$
stage3 $(8984,8984,8984)$ $(2702,2702,2702)$ $(1304,1304,1304)$
总 $(763,763,763,763,763)$

训练过程与测试结果

每个阶段自动调参，得到三个分类器。

第一阶段测试结果：测试 $5554$ 组数据，准确率 $80.48\%$

第二阶段测试结果：测试 $4282$ 组数据，准确率 $87.34\%$

第三阶段测试结果：测试 $3912$ 组数据，准确率 $77.40\%$

总测试结果：准确率 $58.19\%$

这么低的准确率不在预想范围中，却非常合理——每一阶段都只有 $80\%$ 的准确率，乘起来准确率不就很低了嘛！

jieba+nltk

背景

后来啊，我了解到 python 有一个 nltk 模块，即 Natural Language Toolkit，正如其名，里面有很多很多自然语言处理相关工具，诸如朴素贝叶斯、逻辑回归、随机梯度下降、支持向量机等等。但是 nltk 中只有英文分词工具，中文分词还是得靠 jieba 分词。

记录

分类器 准确率
$\text{Naive Bayes}$ $54.00\%$
$\text{Multinomial Naive Bayes}$ $70.99\%$
$\text{Bernoulli Naive Bayes}$ $70.45\%$
$\text{Logistic Regression}$ $71.99\%$
$\text{Stochastic Gradient Descent}$ $71.53\%$
$\text{SVC}$ $66.45\%$
$\text{Linear SVC}$ $67.45\%$
$\text{NuSVC}$ $68.86\%$

综合考量

综合所有得到的分类器，加权对文本进行分类，具体方式是：对于一句评论，采用不同的分类器进行分类，以该分类器的准确度作为权重给对应的类别加权，最后取权重最大者为该评论的类别。

最终的总准确率约为：$\color{blue}{72.36\%}$
